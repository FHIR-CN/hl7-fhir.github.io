EHRS FM Record Lifecycle Event - Audit Event (Profile)

Defines the elements to be supported within the AuditEvent resource in order to conform with the Electronic Health Record System Functional Model Record Lifecycle Event standard
Profiles:
EHRAuditEventDefines the elements to be supported within the AuditEvent resource in order to conform with the Electronic Health Record System Functional Model Record Lifecycle Event standard
Extensions:
auditevent-ehrs-rle-locationWhere the activity occurred, if relevant :

The phyiscal location of the event (e.g. Building, room, etc.).

auditevent-ehrs-rle-reasonReason activity is occurring :

The reason that the activity was taking place.

auditevent-ehrs-rle-policyPolicy or plan th eactivity was defined by :

Policy or plan the event was defined by. Typically, a single event may have multiple applicable policy documents, such as patient consent, guarantor funding, etc.
